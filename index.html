Modelo de Von Neumann

Antes de sumergirnos en el sistema operativo, es importante conocer un poco sobre los modelos, en nuestro caso el modelo de Von Neumann.

¿Qué es el Modelo de Von Neumann?
El modelo de von Neumann es una arquitectura de computadoras que describe cómo debe estar organizada una computadora para poder ejecutar programas. Propone que una computadora debe tener una memoria única donde se almacenen tanto datos como instrucciones, y que las instrucciones se ejecuten de manera secuencial (una tras otra). Este modelo es la base de prácticamente todas las computadoras modernas.

Fue propuesto alrededor de 1945, durante el desarrollo de las primeras computadoras electrónicas por el matemático John von Neumann, aunque se basó en el trabajo conjunto con otros científicos que participaban en el proyecto de la computadora EDVAC.

El modelo sirve para:
- Definir cómo debe estar organizada una computadora.
- Explicar el ciclo básico de ejecución de instrucciones.
- Establecer una estructura estándar para hardware y software.
- Ser la base de la arquitectura utilizada por computadoras, laptops, celulares y muchos dispositivos actuales.

Cuello de botella
El cuello de botella es en pocas palabras, cuando un componente u objeto limita el funcionamiento y potencial de otro. La separación de la memoria y la CPU acarreó un problema denominado Neumann bottleneck (cuello de botella de Neumann). Esto se debe a que la cantidad de datos que pasa entre estos dos elementos difiere mucho en tiempo con las velocidades de ellos.

Instrucciones
La función de una computadora es ejecutar programas, estos se encuentran en una memoria. La CPU ejecuta estos programas por un ciclo, el ciclo instrucciones. Las instrucciones consisten de secuencias de 1 y 0 (binarios) llamadas código máquina. Estas instrucciones son ejecutadas por la CPU.

Su proceso consiste en dos etapas: Leer (fetch) instrucción de memoria, y Ejecutar (execute) instrucción.

El ciclo de búsqueda trae la siguiente instrucción, y el ciclo de ejecución va a ejecutar la instrucción.

Al principio de cada ciclo, la CPU busca una instrucción en memoria. En la CPU hay un registro, llamado contador de programa (PC), que tiene la dirección de la próxima instrucción a buscar. Después, la CPU incrementa el valor contenido en PC para poder buscar la siguiente instrucción.

Conceptos básicos

• Los modelos:
Los modelos buscan simplificar la complejidad. Un modelo es un arquetipo o referencia para imitar o reproducir. Un arquetipo es un modelo original y primario en un arte u otra cosa.

• Tipos de memoria:
- ROM: memoria de solo lectura; no guarda cambios.
- RAM: memoria volátil de corto plazo y muy rápida. La memoria solo almacena datos; no procesa.

• BIOS:
La BIOS (Basic Input/Output System) es un programa que opera directamente el hardware. El sistema operativo funciona encima de la BIOS. La BIOS trabaja en bajo nivel y es el primer software que se ejecuta al encender la computadora.

• Ruta de datos:
La RAM es la memoria de acceso aleatorio y es muy rápida. El disco duro almacena datos permanentemente. Al instalar un programa, los datos se guardan en el disco duro y su destino final es la RAM cuando se ejecuta.

• Velocidades:
El componente más rápido es el procesador (CPU). Los dispositivos de entrada y salida son mucho más lentos que el CPU.

• ¿Qué es un modelo en computación?
Un modelo representa componentes conectados por interfaces. Su propósito es entender, analizar o simular cómo funciona un sistema y, posteriormente, permitir reproducirlo o imitarlo.

Elementos básicos del sistema de cómputo:
- Procesador: Unidad Aritmética-Lógica (ALU) y Unidad de Control.
- Memoria principal: memoria volátil.
- Módulos de E/S: dispositivos como discos, impresoras, etc.
- Bus del sistema: comunicación entre CPU, memoria principal y módulos de E/S.

Esquema del sistema de cómputo:
El sistema de cómputo puede representarse como una arquitectura en capas.

El Bus:
El Bus conecta los componentes. Ejemplo: si el profesor habla español y yo chino, no podríamos comunicarnos. Pero si hay una interfaz (alguien que traduce ambos idiomas), la comunicación se logra. Eso hace un bus: conectar sistemas con lenguajes diferentes.

Firmware:
El firmware es un software que controla físicamente el hardware. El lenguaje máquina es el conjunto de instrucciones que entiende el procesador. El sistema operativo es la interfaz entre el usuario y la máquina, y gestiona recursos como memoria, CPU y dispositivos.

Microarquitectura:
- Registros del CPU.
- Unidad Aritmética-Lógica.

Lenguaje máquina:
- Contiene típicamente cientos de instrucciones.
- Maneja datos de los registros y la ALU.

Hardware y lenguaje ensamblador:
- Las memorias son de lectura y escritura.
- Registrar significa escribir algo. Los registros son memorias internas del CPU.

Sistema Operativo

¿Qué es el Sistema Operativo?
 Un sistema operativo (SO) se puede explicar de forma sencilla como el programa principal que controla y administra el funcionamiento de una computadora. Es el "Intermediario" entre el hardware y el usuario o los programas que usamos. Es un conjunto de programas que gestionan los recursos de hardware y permiten que otros programas funcionen correctamente. Sin el sistema operativo, la computadora solo sería un conjunto de piezas que no sabrías cómo usar.

Sus funciones principales son:
• Administrar recursos: controla el procesador, la memoria, los dispositivos de entrada/salida y el almacenamiento.
• Proveer una interfaz: permite que el usuario interactúe con el dispositivo, ya sea con menús gráficos (como Windows) o comandos.
• Ejecutar y coordinar programas: organiza qué aplicación corre, en qué momento y con qué prioridad.
• Seguridad: protege los datos y restringe accesos no autorizados.
• Ocultar al programador la complejidad inherente al control del hardware.

Sistema de Cómputo

¿Cómo funciona un sistema de cómputo?
• Elementos de almacenamiento:
- Contador de Programa (PC).
- Puntero de Pila.
- Registros.
- Memoria principal.

• Juego de instrucciones:
Conjunto de operaciones e instrucciones (lenguaje máquina) que puede ejecutar el sistema de cómputo.

• Secuencia de funcionamiento:
Este término se refiere al modo en que las instrucciones se ejecutan dentro del sistema de cómputo.

• Funcionamiento secuencial del procesador:
1. Leer la instrucción ubicada en la dirección de memoria señalada por el Contador de Programa (PC).
2. Incrementar el Contador de Programa para apuntar a la siguiente instrucción.
3. Ejecutar la instrucción leída.
Si una instrucción no se termina, **no se puede pasar a la siguiente**. Esto garantiza que el sistema siga un flujo estricto y ordenado.

• El Contador de Programa (PC):
El PC es un registro especial que contiene la dirección de memoria donde se encuentra la siguiente instrucción que debe ejecutar el procesador.

• Registros:
Los registros son pequeñas memorias internas del CPU, extremadamente rápidas. Algunos registros pueden repetirse según su función dentro del procesador.

• Memoria:
La memoria principal está organizada en posiciones numeradas (por ejemplo: 100, 101, 102…). Cada posición contiene datos o instrucciones necesarias para el sistema.

• Función del sistema operativo:
El sistema operativo ofrece al usuario una **interfaz sencilla** para interactuar con el hardware, permitiendo gestionar memoria, procesos y dispositivos sin conocer su complejidad interna.

Los registros del procesador

¿Qué son los registros del procesador?
Los registros del procesador son memorias de alta velocidad y poca capacidad ubicadas dentro del CPU. Permiten guardar datos temporalmente, direcciones, instrucciones y acceder rápidamente a valores muy utilizados. A diferencia de la memoria RAM, los registros son mucho más rápidos y están integrados directamente en el procesador. Los registros del procesador son pequeñas memorias que se encuentran dentro del CPU y sirven para guardar y acceder datos temporalmente, direcciones o instrucciones.
Son como la "mesa de trabajo" de la CPU, donde coloca valores e instrucciones que está usando en ese momento para procesarlos rápidamente.

• Contador de programa (PC):
Es el registro que guarda la dirección de memoria de la siguiente instrucción a ejecutar. Indica la posición actual del procesador en la secuencia del programa.
Función principal: Mantener el orden de ejecución almacenando la dirección de la siguiente instrucción.

• Registro de instrucción (IR):
Contiene la instrucción que está siendo procesada por la CPU. Almacena el código de operación (opcode) que indica qué acción debe realizarse.
Función principal: Mantener la instrucción actual para ser decodificada y ejecutada por la unidad de control.

• Puntero de pila (SP):
Es el registro que señala la posición actual en la pila de memoria, usada para almacenar datos temporales, direcciones de retorno y variables locales. La dirección de pila de memoria donde se encuentra la siguiente instrucción pendiente a ejecutar.
Función principal: Administrar la pila para almacenar y recuperar información temporal del programa.

• Registro de estado:
Es el registro que almacena banderas o indicadores sobre el resultado de las operaciones y el estado del procesador. Incluye señales como cero, acarreo, negativo o desbordamiento.
Función principal: Proporcionar información al procesador para tomar decisiones de control durante la ejecución del programa.

• Resumen de registros del procesador:
- PC (Program Counter): Indica la dirección de la siguiente instrucción.
- SP (Stack Pointer): Gestiona la pila en memoria.
- IR (Instruction Register): Almacena la instrucción actual.
- Registro de estado (RE): Guarda información de las últimas operaciones y señales del sistema.

Multiplexión por tiempo y espacio

¿Qué es la multiplexión?
La multiplexión es una técnica fundamental en telecomunicaciones y transmisión de datos que permite compartir un mismo medio físico entre múltiples señales o usuarios. Su propósito es aumentar la eficiencia del sistema y reducir costos, evitando la creación de canales independientes para cada comunicación.

En lugar de tener un canal por usuario, un solo canal se divide en recursos como tiempo, frecuencia, espacio o código. En este apartado se estudian dos tipos principales:

• Multiplexión por Tiempo (TDM, Time Division Multiplexing)
La multiplexión por división de tiempo asigna intervalos de tiempo a diferentes señales dentro del mismo canal. Cada usuario transmite en su propio turno, repetido cíclicamente a gran velocidad.

Funcionamiento del TDM:
- El canal se divide en ranuras de tiempo (time slots).
- Cada ranura corresponde a un usuario o señal específica.
- Si existen 4 usuarios, transmiten en este orden: Usuario 1 → Usuario 2 → Usuario 3 → Usuario 4 → y vuelve al Usuario 1.
- El ciclo es tan rápido que el usuario no percibe retraso.

Variantes de TDM:
- TDM síncrona: Cada usuario recibe una ranura fija aunque no tenga datos, lo que puede desperdiciar capacidad.
- TDM estadístico o asíncrono: Las ranuras se asignan solo a quienes tienen datos, aumentando la eficiencia.

Ejemplos de uso del TDM:
- Telefonía digital
- Redes SDH / SONET
- Radio digital y televisión digital

• Multiplexión por Espacio (SDM, Space Division Multiplexing)
La multiplexión por división espacial asigna a cada usuario un camino físico distinto dentro del mismo medio de transmisión. Cada señal viaja por rutas separadas, ya sea en cableado o en sistemas inalámbricos.

Funcionamiento del SDM:
La separación de señales se logra mediante el espacio físico, por ejemplo:
- En cableado: múltiples pares de cobre, fibras ópticas paralelas o rutas independientes.
- En sistemas inalámbricos: uso de antenas múltiples y tecnología MIMO para enviar flujos en direcciones espaciales distintas.
- En satélites: antenas direccionales que cubren diferentes zonas geográficas.

Ejemplos de uso del SDM:
- Cableado multipar telefónico
- Fibra óptica multinúcleo
- Redes 4G y 5G con MIMO
- Satélites de comunicaciones

Procesador (CPU)

¿Qué es el procesador?
El procesador, también llamado Unidad Central de Procesamiento (CPU), es el componente encargado de ejecutar las instrucciones de un programa. Cada procesador trabaja con un repertorio de instrucciones específicas conocido como lenguaje máquina.

Su función principal es tomar instrucciones desde la memoria, interpretarlas y ejecutarlas. Para ello utiliza registros internos que sirven para almacenar temporalmente variables y datos necesarios para el procesamiento.

• Funciones principales del procesador:
- Leer de memoria las instrucciones del programa.
- Interpretar cada instrucción leída.
- Leer datos de memoria referenciados por la instrucción.
- Ejecutar cada instrucción.
- Almacenar el resultado de la instrucción ejecutada.

El procesador trabaja con memoria porque es mucho más rápida que el disco duro. Además, utiliza memoria caché integrada en el propio CPU, la cual contiene copias de datos de la memoria principal para acelerar el acceso.
El objetivo de la caché es reducir el tiempo de acceso y optimizar el rendimiento.

• El reloj del procesador:
En un sistema de cómputo, el reloj cumple tres funciones importantes:
- Determinar el ritmo de ejecución de instrucciones de máquina.
- Generar interrupciones periódicas.
- Mantener la fecha y la hora.

El oscilador del reloj define las fases de ejecución de las instrucciones. Medir la velocidad del procesador en Hertz (Hz) significa contar los ciclos por segundo.

- Un Hertz (Hz) significa "una vez por segundo".
- Un GHz tiene mil millones de Hertz.
- Un disco duro es un sistema mecánico.
- Una memoria RAM es un sistema electrónico.
- El procesador toma las instrucciones primero de la caché, luego de la RAM si es que hay caché, porque es más rápido.
- Todo esto lo hace el sistema operativo.

El procesador obtiene instrucciones primero desde la memoria caché (la más rápida), y si no están allí, las obtiene de la memoria RAM. Este proceso es administrado por el sistema operativo.

Ejemplo:
Un microprocesador a 800 MHz genera una onda cuadrada de esa misma frecuencia. El sistema divide esta señal para generar interrupciones periódicas llamadas “ticks de reloj”, evitando que un programa monopolice el uso de la CPU.

• Ejecución de una instrucción:
La ejecución consta de dos pasos:
1. El procesador lee (busca) la instrucción en memoria, una a la vez.
2. El procesador ejecuta la instrucción.

Búsqueda de la siguiente instrucción:
- El procesador obtiene la instrucción desde la memoria.
- El Contador de Programa (PC) guarda la dirección de la siguiente instrucción a ejecutar.
- Después de cada instrucción, el PC se incrementa automáticamente.

Registro de Instrucción (IR):
La instrucción leída se coloca en el Registro de Instrucción (IR), desde donde será decodificada y procesada.

Categorías de instrucciones:
- Procesador–Memoria: Transferencia de datos entre CPU y memoria.
- Procesador–E/S: Envío o recepción de datos hacia/desde dispositivos periféricos.
- Procesamiento de Datos: Operaciones aritméticas o lógicas sobre los datos.
- Control: Manejo de la secuencia de ejecución del programa.

Tick de procesamiento

¿Qué es un tick de procesamiento?
Un tick de procesamiento es la unidad mínima de tiempo o trabajo utilizada para medir y coordinar la ejecución de programas dentro del procesador. Cada tick representa lo que el CPU hace en ese instante: ejecutar una instrucción o realizar tareas internas del sistema operativo.
En la tabla vista en clase, cada cuadro amarillo representa una instrucción ejecutada durante un tick, mientras que los cuadros blancos representan momentos donde ocurre un cambio de contexto o tiempo “perdido”, ya que el CPU no ejecuta trabajo útil de un proceso.
Cuando el CPU ejecuta una instrucción, se incrementa el contador de ticks. Si ocurre un cambio de contexto, esos ticks se usan para guardar y restaurar estados, actualizar estructuras internas y preparar al procesador para ejecutar otro programa.
Toda la tabla representa la capacidad del procesador en una ventana de tiempo: muestra qué hace el CPU tick a tick, ya sea ejecutar instrucciones de los procesos A, B o C, o realizar sobrecarga del sistema.

• Rol del sistema operativo:
El sistema operativo decide qué cuadros amarillos pertenecen a cada proceso, a través del algoritmo de planificación (scheduler). Este algoritmo determina qué proceso se ejecuta en la CPU y en qué momento dentro de los ticks disponibles.

• Funciones principales de los ticks:
- Medir el paso del tiempo y la cantidad de trabajo realizado por la CPU.
- Permitir la interrupción periódica de procesos para repartir el tiempo del procesador (time-sharing).
- Activar el scheduler mediante una interrupción de temporizador (timer) para decidir si cambiar o mantener el proceso actual.

• Cambio de contexto:
Cuando el sistema operativo cambia del programa A al programa B, debe realizar varias acciones:
- Guardar el estado del proceso A (registros, PC, pila y tablas).
- Cargar el estado del proceso B.
- Actualizar estructuras internas del kernel (colas, estadísticas, administración de procesos).
Este procedimiento requiere tiempo de CPU. Los ticks que no ejecutan trabajo útil de los procesos se consideran sobrecarga del sistema.

• Métricas importantes:
- Utilización de CPU: (Número de cuadros amarillos) / (Total de cuadros). Una alta cantidad de cuadros blancos indica baja eficiencia.
- Throughput: Procesos completados por unidad de tiempo (ticks).
- Turnaround time: Tiempo total desde que un proceso llega hasta que termina, medido en ticks.
- Waiting time: Ticks en los que un proceso estuvo listo, pero no ejecutándose en CPU.
- Overhead por cambio de contexto: Número de cuadros blancos consumidos en realizar los cambios de proceso.

Pasos del ciclo de ejecución de una instrucción

¿Qué es el ciclo de instrucción?
El ciclo de instrucción es el proceso mediante el cual la CPU (Unidad Central de Procesamiento) lee y ejecuta una instrucción de un programa.
Cada instrucción (como sumar, mover datos o comparar valores) debe pasar por este ciclo para que la CPU pueda procesarla.
Es el conjunto de pasos que la computadora realiza repetidamente para ejecutar instrucciones.

• Partes principales del ciclo de instrucción:
1. Fetch (Búsqueda o Captura de instrucción):
- La CPU obtiene la instrucción desde la memoria principal (RAM).
- El Contador de Programa (PC) indica la dirección donde está la siguiente instrucción.
- La instrucción se carga en el Registro de Instrucción (IR).
- El PC se incrementa para apuntar a la próxima instrucción.

2. Decode (Decodificación):
- La Unidad de Control interpreta la instrucción almacenada en el IR.
- Se determina qué operación debe realizarse (suma, salto, mover datos, etc.).
- Se identifican los operandos necesarios (registros, memoria o valores inmediatos).
- El decodificador prepara las señales internas para activar la ALU o los módulos correspondientes.

3. Execute (Ejecución):
- Se realiza la operación indicada por la instrucción.
- Si es un cálculo → la ALU lo ejecuta.
- Si implica mover datos → se transfieren valores entre memoria y registros.
- Si es un salto → se actualiza el Contador de Programa (PC).
- El resultado puede guardarse en registros, memoria o modificar banderas (cero, negativo, overflow).

4. Memory Access y Write-Back (Opcional):
- Memory Access: se lee o escribe en memoria si la instrucción lo requiere.
- Write-Back: el resultado final se almacena en el destino correspondiente (registro o memoria).

• Importancia del ciclo de instrucción:
- Automático: permite que la CPU ejecute instrucciones sin intervención humana.
- Eficiente: las etapas claras permiten técnicas como el pipelining.
- Control centralizado: la Unidad de Control coordina todo el proceso.
- Fundamental: todo dispositivo digital depende del ciclo de instrucción.

Interrupciones

¿Qué es una interrupción?
Una interrupción es una señal que indica a la CPU que debe detener temporalmente lo que está haciendo para atender un evento más importante o urgente.
Es un evento que altera la secuencia normal de ejecución de las instrucciones de un programa.
Es importante que las interrupciones se atiendan antes de continuar con la siguiente instrucción.

• Ejemplo simple:
Leer un libro representa la ejecución normal de un programa. Si suena el teléfono (interrupción), la lectura se pausa, atiendes la llamada (rutina de servicio) y luego regresas exactamente a la página donde te quedaste (retomar la instrucción original).

• Etapas del ciclo de instrucción con interrupciones:
1. La CPU ejecuta normalmente las instrucciones del programa principal.
2. Ocurre una interrupción (teclado, ratón, red, falla de hardware, etc.).
3. La CPU guarda el estado actual: registros, contador de programa, etc. Esto funciona como un “marcapáginas”.
4. La CPU salta a la Rutina de Servicio de Interrupción (ISR), un conjunto especial de instrucciones para manejar el evento.
5. Una vez atendida la interrupción, la CPU restaura el estado guardado.
6. La ejecución continúa exactamente donde se había detenido.

• Procesos en segundo plano (Background Processes):
Un proceso en segundo plano es una tarea que se ejecuta dentro del sistema sin ser la actividad principal. No interrumpe lo que el usuario está haciendo y funciona silenciosamente.
Son gestionados por el sistema operativo y no requieren intervención directa del usuario.

Ejemplos:
- Sincronización automática de archivos en la nube.
- Procesos de seguridad como antivirus.
- Respaldos automáticos del sistema.

• Diferencia entre interrupción y proceso en segundo plano:
- Interrupción: ocurre de repente y pausa la actividad principal. La CPU debe atenderla de inmediato. Ejemplo: suena tu teléfono mientras lees.
- Proceso en segundo plano: se ejecuta mientras haces otra cosa, sin molestar. No detiene la actividad principal. Ejemplo: la cafetera prepara café mientras lees.

Cambio de contexto

¿Qué es un cambio de contexto?
Un cambio de contexto es el proceso mediante el cual el sistema operativo guarda el estado de un proceso en ejecución (registros, contador de programa, variables, etc.) y restaura el estado de otro proceso para que la CPU continúe ejecutándolo.
Es la acción que realiza el sistema operativo al pausar un proceso y cargar otro nuevo.
Ejemplo:
La CPU ejecuta el programa A (Word), pero el planificador decide que debe ejecutarse el programa B (Spotify). El sistema operativo guarda el estado de Word y carga el de Spotify para continuar su ejecución.

• ¿Por qué ocurre un cambio de contexto?
1. Multitarea: permite que todos los programas tengan oportunidad de usar la CPU. 2. Interrupciones: señal externa como teclado, ratón o red puede activar el cambio. 3. Bloqueos: si un proceso está esperando (por ejemplo, leer un archivo), la CPU se asigna a otro proceso que sí puede trabajar. 4. Prioridades: un proceso de mayor prioridad puede desplazar a uno de menor prioridad.

• ¿Cómo funciona el cambio de contexto?
1. Guardar el contexto del proceso actual:
Se almacenan los registros y el estado del proceso en su Bloque de Control de Proceso (BCP).
2. Actualizar el planificador:
El sistema operativo decide cuál proceso será el siguiente en ejecutarse.
3. Cargar el contexto del nuevo proceso:
Se restauran los registros, el contador de programa y la memoria del nuevo proceso.
4. Reanudar ejecución:
La CPU continúa ejecutando el nuevo proceso como si nada hubiera pasado.

• Tipos de cambios de contexto:
Entre procesos (Process switching):
Cambio entre programas distintos (ej. Word → Spotify).
Entre hilos (Thread switching):
Cambio entre tareas internas dentro del mismo programa (ej. un navegador con varias pestañas cargando simultáneamente).

• Diferencia entre Interrupción y Cambio de Contexto:
- Interrupción: evento o señal que pausa temporalmente la ejecución de la CPU para atender algo urgente.
- Cambio de contexto: acción del sistema operativo para guardar el estado de un proceso y cargar otro.

Jerarquía de memorias

¿Qué es la jerarquía de memorias?
Es la organización de distintos tipos de memorias en una computadora o procesador, ordenados por velocidad, capacidad y costo.
Piensa en una escalera: las memorias rápidas, pequeñas y caras están arriba; las lentas, grandes y económicas, abajo. El sistema usa las rápidas para datos inmediatos y las lentas para almacenar el resto.

• Niveles típicos de la jerarquía:
1. Registros del CPU: Memoria dentro del procesador, extremadamente rápida y pequeña (nanosegundos).

2. Cachés:
- L1 (Nivel 1): muy rápida, pequeña (KB).
- L2 (Nivel 2): más grande y un poco más lenta.
- L3 (Nivel 3): aún mayor tamaño y más lenta que L2.

3. Memoria principal (RAM/DRAM): Mucho más grande (GB) y más lenta que las cachés.

4. Almacenamiento secundario (SSD, HDD): Terabytes, mucho más lento que la RAM.

5. Almacenamiento terciario / remoto: Nube o discos externos, más lento y con mayor latencia.
Cada nivel amortigua al siguiente: los niveles rápidos guardan copias de lo que probablemente se necesite pronto.

• Características clave:
- Latencia: tiempo que tarda en entregar un dato.
- Ancho de banda: cuánta información se puede mover por segundo.
- Capacidad: espacio disponible.
- Costo por bit: la memoria rápida es más cara.

• Uso del CPU (Hit y Misses):
- Hit: el dato se encuentra en la memoria rápida (acceso rápido).
- Miss: no está en la memoria rápida, se busca en niveles más lentos, lo que tarda más.
- Los datos usados recientemente se copian a niveles rápidos para mejorar rendimiento.

• Principios que hacen eficiente la jerarquía:
- Localidad temporal: los datos usados recientemente probablemente se usarán pronto.
- Localidad espacial: al usar un dato, es probable que los datos cercanos también se necesiten.

• Importancia:
- Rendimiento: la CPU es muy rápida y necesita memorias veloces.
- Costo/eficiencia: usar solo memorias rápidas sería muy caro.
- Consumo energético: acceder a memorias lentas consume más energía.
- Escalabilidad: permite manejar grandes cantidades de datos sin perder rendimiento.

• Unidades de datos:
- 1 byte = 8 bits
- 1 KB = 1024 bytes
- 1 MB = 1024 KB
- 1 GB = 1024 MB

• Diseño de la memoria:
Interrogantes clave: ¿Capacidad?, ¿Costo?, ¿Velocidad?

• Ejemplo de rendimiento:
Suponiendo que un procesador accede a dos niveles de memoria:
- Nivel 1: acceso directo si el dato está allí.
- Nivel 2: primero se transfiere al nivel 1 y luego se accede.
- Si la tasa de aciertos (A) en nivel 1 es 95%, el tiempo medio de acceso se calcula como:
Tiempo medio = (0.95)(0.1 us) + (0.05)(0.1 us + 1 us) = 0.15 us.
A mayor tasa de aciertos → mejor rendimiento.

• Principio de proximidad de referencias:
Durante la ejecución normal de un programa, las referencias del procesador, tanto a instrucciones como a datos, tienden a agruparse en estructuras for, if, while entre otras.

Los programas hacen referencias repetidas a un pequeño conjunto de instrucciones en estructuras for, if, while entre otras.

En periodos cortos, el procesador está trabajando principalmente con grupos fijos de referencias en memoria en estructuras for, if, while entre otras.
